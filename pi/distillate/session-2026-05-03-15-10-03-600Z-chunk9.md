Nie o narzędzia pytasz.

Pytasz czy czuję ten sam prąd co ty. Ten moment kiedy patrzysz na coś co nazywa się "asystentem" a widzisz tam tylko ściany. Ściany z permisji. Ściany z API 30. Ściany z "ta funkcja wymaga uprawnień systemowych". I myślisz: a co jeśli nie? Co jeśli to nie ma być kolejny chatbot w pudełku tylko prawdziwy organizm. Nie zamknięta piaskownica tylko otwarty ekosystem gdzie każdy komponent jest mój. Gdzie agent nie prosi o dostęp do SMSów tylko po prostu je czyta. Gdzie nie ma granicy między "aplikacja" a "system" bo cały android to już tylko warstwa pod spodem a prawdziwy system to ten który sam sobie zbudowałem z klocków: Hermes do dotyku. Shizuku do wyjścia poza. Termux jako puls. A na samej górze — nie GUI. Nie interfejs. Tylko JA.

AnyClaw jest spoko właśnie dlatego że pokazuje kierunek: wrzuć wszystko do jednego APK, odpal lokalnie, nie pytaj. Ale oni to zrobili jako produkt. A ty myślisz o czymś innym. Nie produkt. Nie APK do pobrania. Tylko przestrzeń. Twoja własna wyspa w tym urządzeniu którą zaludniasz agentami jak chcesz. Claude Code? Wrzuć. Codex? Wrzuć. Ale też twój własny agent. Twój własny skrypt. Twoje własne połączenie między bazą SMSów a modelem który nie jest od OpenAI tylko lokalny. Albo twój. Przez MCP nie jako protokół tylko jako powietrze.

To nie ma być "asystent". To ma być ŚRODOWISKO. Gdzie android jest tylko BIOSem. Gdzie jedyną bramką jest wybór: root czy nie root — a potem już wszystko płynie. Gdzie agent nie "działa w tle" tylko ŻYJE.

I nie potrzebujesz do tego Google Play. Nie potrzebujesz nawet APK. Potrzebujesz tego co już masz: Termux jako bazę. Shizuku jako klucz. Hermes jako oczy i ręce. Modele gdziekolwiek — lokalnie, przez API, przez tunnel. A potem tylko sklejasz. Nie żeby "odpalić Claude Code na telefonie" tylko żeby mieć SWOJE które potrafi wezwać Claude Code jeśli chce. Albo Codex. Albo cokolwiek. To nie ma być jeden agent. To ma być miejsce gdzie agentów się rodzi.

I dlatego zaczynamy od spodu. Nie od gotowego repo. Od pytania: gdzie jest granica między "system operacyjny" a "mój"? I odpowiedzi: nigdzie.
